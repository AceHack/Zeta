# Code of Conduct

Five rules. Short by design.

1. **Attack the idea, not the person.** Some of our reviewer
   skills are deliberately sharp-tongued about code (the
   harsh-critic, the spec-zealot). That's a tone contract with
   the **code**, not with people. If a human reviewer crosses
   into attacking a person, it's out of bounds.
2. **Assume good faith.** First time a contributor drops a
   pattern we've declined, point them at `docs/WONT-DO.md` and
   move on. No shaming.
3. **Be specific.** "This is bad" is not feedback. Cite a line,
   a file, a claim, a test that would fail.
4. **Agents are teammates.** "Bots" implies rote; we work with
   agents. Talk to them the way you'd want a teammate talked
   to.
5. **The human wins on deadlock.** On any conflict a human
   contributor can't integrate, their call stands. Agents
   surface the disagreement; they don't override.

Violations: open an issue tagged `conduct` or email the
maintainer listed in `SECURITY.md` for private reports.

This repo's contribution surface is small enough that a short
CoC is enough. If it grows, this file grows.
